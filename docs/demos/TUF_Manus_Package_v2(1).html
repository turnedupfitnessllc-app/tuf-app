Critical — File Co-Location Rule
TUF_App_v2.html and TUF_DATA.js must be in the exact same folder. The app loads the data file via <script src="TUF_DATA.js"></script> — a relative path. 04
ARCHITECTURE + DATA FLOW
Phase 1 Architecture — 100% Static, No Backend Required
The app is pure HTML/CSS/JS. No server, no database, no API keys required for Phase 1. Deploy to any static host. The TUF_DATA.js file is a JavaScript object — the app loads it as a regular script tag. Admin changes are saved to localStorage and the member app listens for those changes via a storage event and re-renders automatically. Phase 2: Supabase replaces localStorage. Wilfred builds the API layer. Manus wires the backend. Not yet.
/* How it loads — browser executes in this order: */ 1. Browser loads TUF_App_v2.html 2. Hits <script src="TUF_DATA.js"> → loads data layer globally as window.TUF_DATA 3. Hits inline <script> → app functions defined 4. renderHomeScreen() fires → reads TUF_DATA → renders HOME screen 5. User taps nav → showScreen() → reads TUF_DATA → renders that screen /* Admin sync: */ Admin edits TUF_Admin.html → saves to localStorage('TUF_EXERCISES' etc) Member app window.addEventListener('storage') → detects change → re-renders
